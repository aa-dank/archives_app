Start with the Results table when you want the most likely matching files. Each row is one file-level result grouped by file hash, so duplicate copies of the same file are combined. Use Primary Location to open or review the file, and expand the row details when the same file appears in additional indexed locations.

Match shows whether the query was found in the filename, path, extracted document text, or a combination. Snippet appears when document text matched and gives a short preview of the matching text. Expanded row details include all indexed locations, searchability status, and file hash information.

Coverage explains how much of the selected scope could actually be searched by content. A content search with low coverage can miss relevant files whose text has not been extracted, failed extraction, has not been chunked, or is in a format that is not useful for text extraction.

The Excel workbook contains up to the top {{ search.excel_file_limit }} file-level results and is intended for further filtering, sorting, and searching. It includes a Results sheet for the grouped matches, a Locations sheet with all indexed locations for exported files, and a Coverage sheet with the search settings and coverage counts.
